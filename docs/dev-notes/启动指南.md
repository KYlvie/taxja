## 第一步：启动 Docker Desktop

### Windows 用户

1. **打开 Docker Desktop**
   - 在开始菜单搜索 "Docker Desktop"
   - 点击启动
   - 等待 Docker 图标变绿（表示已启动）

2. **验证 Docker 是否运行**
   ```bash
   docker ps
   ```
   如果看到表格（即使是空的），说明 Docker 已启动。

### 如果没有安装 Docker Desktop

下载并安装：https://www.docker.com/products/docker-desktop/

## 第二步：启动项目（三种方式）

### 🎯 方式一：完整 Docker 模式（最简单，推荐）

所有服务都在 Docker 中运行，一键启动。

```bash
# 在项目根目录执行
docker-compose up -d
```

等待 1-2 分钟，然后访问：
- **前端**: http://localhost:5173
- **后端 API**: http://localhost:8000
- **API 文档**: http://localhost:8000/docs

**初始化数据库**（首次运行需要）：
```bash
# 进入后端容器
docker exec -it taxja-backend bash

# 运行数据库迁移
alembic upgrade head

# 加载演示数据
python scripts/seed_demo.py

# 退出容器
exit
```

---

### 🔧 方式二：混合模式（推荐开发调试）

基础设施用 Docker，后端和前端在本地运行。

#### 2.1 启动基础设施（数据库、Redis、MinIO）

```bash
docker-compose up -d postgres redis minio
```

#### 2.2 启动后端

**新开一个终端窗口**：

```bash
cd backend

# 创建 .env 文件
echo SECRET_KEY=dev-secret-key-change-in-production > .env
echo ENCRYPTION_KEY=dev-encryption-key-change-in-production >> .env
echo POSTGRES_SERVER=localhost >> .env
echo POSTGRES_USER=taxja >> .env
echo POSTGRES_PASSWORD=taxja_password >> .env
echo POSTGRES_DB=taxja >> .env
echo POSTGRES_PORT=5432 >> .env
echo REDIS_HOST=localhost >> .env
echo REDIS_PORT=6379 >> .env
echo MINIO_ENDPOINT=localhost:9000 >> .env
echo MINIO_ACCESS_KEY=minioadmin >> .env
echo MINIO_SECRET_KEY=minioadmin >> .env

# 安装依赖（如果还没安装）
pip install -r requirements.txt

# 运行数据库迁移
alembic upgrade head

# 加载演示数据（可选）
python scripts/seed_demo.py

# 启动后端服务器
uvicorn app.main:app --reload --port 8000
```

后端会在 http://localhost:8000 运行

#### 2.3 启动前端

**再开一个新终端窗口**：

```bash
cd frontend

# 启动开发服务器
npm run dev
```

前端会在 http://localhost:5173 运行

#### 2.4 启动 Celery Worker（可选，用于 OCR 处理）

**再开一个新终端窗口**：

```bash
cd backend
celery -A app.celery_app worker --loglevel=info
```

## 常见问题解决

### ❌ 问题 1: Docker Desktop 没有启动

**错误信息**: `The system cannot find the file specified`

**解决方案**:
1. 打开 Docker Desktop 应用
2. 等待 Docker 图标变绿
3. 重新运行 `docker-compose up -d`

---

### ❌ 问题 2: 端口被占用

**错误信息**: `port is already allocated`

**解决方案**:
```bash
# 查看哪个进程占用了端口
netstat -ano | findstr :5173
netstat -ano | findstr :8000

# 停止占用端口的进程，或者修改 docker-compose.yml 中的端口
```

---

### ❌ 问题 3: 数据库连接失败

**错误信息**: `could not connect to server`

**解决方案**:
```bash
# 检查 PostgreSQL 容器是否运行
docker ps | findstr postgres

# 如果没有运行，启动它
docker-compose up -d postgres

# 等待 10 秒让数据库完全启动
```

---

### ❌ 问题 4: 前端页面空白

**可能原因**: 后端 API 没有启动

**解决方案**:
1. 检查后端是否运行: http://localhost:8000/docs
2. 查看浏览器控制台是否有错误
3. 检查后端日志: `docker-compose logs backend`

---

### ❌ 问题 5: OCR 功能不工作

**可能原因**: Celery Worker 没有启动

**解决方案**:
```bash
# 检查 Celery Worker 是否运行
docker ps | findstr celery

# 查看 Celery 日志
docker-compose logs celery-worker
```

## 停止服务

### 停止所有 Docker 服务
```bash
docker-compose down
```

### 停止并删除所有数据（小心！）
```bash
docker-compose down -v
```

### 只停止某个服务
```bash
docker-compose stop backend
docker-compose stop frontend
```

## 有用的命令

### 查看日志
```bash
# 查看所有服务日志
docker-compose logs -f

# 查看特定服务日志
docker-compose logs -f backend
docker-compose logs -f frontend
docker-compose logs -f postgres
```

### 重启服务
```bash
# 重启所有服务
docker-compose restart

# 重启特定服务
docker-compose restart backend
```

### 进入容器
```bash
# 进入后端容器
docker exec -it taxja-backend bash

# 进入数据库容器
docker exec -it taxja-postgres psql -U taxja -d taxja
```

### 查看容器状态
```bash
# 查看运行中的容器
docker ps

# 查看所有容器（包括停止的）
docker ps -a
```
