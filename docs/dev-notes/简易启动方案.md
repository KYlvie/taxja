# Taxja 简易启动方案（不用 Docker）

## 🎯 最简单的方式：只启动前端

由于 Docker 构建遇到了依赖问题，我们采用更简单的方案：

### 方案：前端 + 模拟数据

这样你可以立即看到完整的界面和交互，虽然数据不会真正保存。

## 🚀 启动步骤

### 步骤 1: 启动前端

```bash
cd frontend
npm run dev
```

等待几秒钟，你会看到：
```
VITE v5.0.11  ready in 1234 ms

➜  Local:   http://localhost:5173/
➜  Network: use --host to expose
```

### 步骤 2: 访问应用

打开浏览器访问: **http://localhost:5173**

## 🎨 你可以看到什么

### ✅ 完整的用户界面

1. **登录页面**
   - 精美的设计
   - 多语言切换（德语/英语/中文）
   - 响应式布局

2. **主界面**（虽然无法真正登录，但可以看到设计）

### ⚠️ 限制

## 💡 如果你想看完整功能

### 选项 A: 修复 Docker 构建（需要时间）

问题是 Python 依赖安装失败。可能的原因：
1. 某些包需要编译（需要 C++ 编译器）
2. 版本冲突
3. 网络问题

**解决方案**：
```bash
# 停止当前构建
docker-compose down

# 清理缓存
docker system prune -a

# 重新构建（可能需要 10-15 分钟）
docker-compose build --no-cache
docker-compose up
```

### 选项 B: 本地安装 PostgreSQL（推荐）

如果你想看完整功能，最简单的方式是：

1. **安装 PostgreSQL**
   ```bash
   # 使用 Chocolatey
   choco install postgresql
   ```

2. **创建数据库**
   ```bash
   psql -U postgres
   CREATE DATABASE taxja;
   CREATE USER taxja WITH PASSWORD 'taxja_password';
   GRANT ALL PRIVILEGES ON DATABASE taxja TO taxja;
   \q
   ```

3. **启动后端**
   ```bash
   cd backend
   pip install -r requirements.txt
   
   # 创建 .env 文件
   echo "SECRET_KEY=dev-secret-key" > .env
   echo "POSTGRES_SERVER=localhost" >> .env
   echo "POSTGRES_USER=taxja" >> .env
   echo "POSTGRES_PASSWORD=taxja_password" >> .env
   echo "POSTGRES_DB=taxja" >> .env
   
   # 运行迁移
   alembic upgrade head
   
   # 加载演示数据
   python scripts/seed_demo.py
   
   # 启动服务器
   uvicorn app.main:app --reload
   ```

4. **启动前端**（新终端）
   ```bash
   cd frontend
   npm run dev
   ```

5. **访问**: http://localhost:5173
