# Taxja 交易系统 - 最终设计方案

## 核心理念：三种输入方式，一个智能系统

用户只需要添加交易，系统自动完成所有分类和识别工作。

---

## 三种输入方式

### 1️⃣ 拍照上传（最简单）📸

**用户操作：**
```
拍照 → 上传 → 完成
```

**系统自动完成：**
- ✅ OCR 识别文字（金额、日期、商家）
- ✅ 自动分类（收入/支出）
- ✅ 识别类别（租金、工资、SVS、餐饮等）
- ✅ 判断可抵税性
- ✅ 提取 VAT 信息
- ✅ 关联房产（如果是租金相关）

**示例：**
```
用户拍照：租金收据
系统识别：
  ✓ 金额：€1,200
  ✓ 日期：2026-03-01
  ✓ 类型：收入
  ✓ 类别：租金收入
  ✓ 可抵税：否
  ✓ 关联房产：Wohnung Mariahilfer Str. 123
```

## 智能定期交易识别 🤖

### 自动模式检测

**触发条件：**
用户连续 2-3 次添加相似交易

**检测逻辑：**
```python
if 相似交易次数 >= 2:
    分析时间间隔
    if 间隔规律（月度/季度/年度）:
        计算置信度
        if 置信度 >= 70%:
            生成智能建议
```

**示例场景：**

#### 场景 1：租金收入
```
用户手动添加：
  2026-01-05: €1,200 租金收入
  2026-02-05: €1,200 租金收入
  2026-03-05: €1,200 租金收入

系统识别：
  ✓ 模式：月度
  ✓ 金额：€1,200
  ✓ 日期：每月 5 号
  ✓ 置信度：95%

系统建议：
"我注意到您每月都有 €1,200 的租金收入，要我自动帮您记录吗？"
```

#### 场景 2：SVS 社保
```
用户手动添加：
  2026-03-15: €2,500 SVS 社保
  2026-06-15: €2,500 SVS 社保

系统识别：
  ✓ 模式：季度
  ✓ 金额：€2,500
  ✓ 日期：每季度 15 号
  ✓ 置信度：90%

系统建议：
"检测到季度 SVS 社保缴纳，要自动记录吗？"
```

#### 场景 3：办公室租金
```
用户手动添加：
  2026-01-01: €800 办公室租金
  2026-02-01: €800 办公室租金
```

## 技术架构

### 后端服务

```
交易输入层
├── OCR 服务（Tesseract + OpenCV）
├── 文件解析服务（PDF/CSV）
└── 合同识别服务（AI 提取关键信息）
         ↓
自动分类层
├── 规则分类器（Rule-based）
├── ML 分类器（scikit-learn）
└── 混合分类器（Hybrid）
         ↓
模式检测层
├── 定期模式检测器
├── 置信度计算
└── 智能建议生成
         ↓
自动化执行层
├── 定期交易生成（Celery）
├── 交易验证
└── 用户通知
```

### 数据流

```
用户输入
    ↓
[Transaction]
    ↓
自动分类 → [category, is_deductible, confidence]
    ↓
模式检测 → [RecurringPattern]
    ↓
用户确认 → [RecurringTransaction]
    ↓
自动生成 → [Transaction (is_system_generated=true)] 🤖
```
