# 交易系统合并方案 - 完全统一

## 目标
将定期交易功能完全合并到交易页面，简化用户操作，统一管理入口。

## 核心理念
**用户只需要知道一件事：记录交易**
- 不需要区分"交易"和"定期交易"两个概念
- 只需选择这笔交易是"单次"还是"定期"
- 系统自动处理后续的生成和管理

## 数据库设计

### Transaction 表扩展
```sql
ALTER TABLE transactions ADD COLUMN:
- is_recurring BOOLEAN DEFAULT FALSE
- recurring_frequency VARCHAR(20) NULL  -- daily, weekly, monthly, quarterly, yearly
- recurring_start_date DATE NULL
- recurring_end_date DATE NULL
- recurring_day_of_month INT NULL
- recurring_is_active BOOLEAN DEFAULT TRUE
- recurring_next_generation_date DATE NULL
- recurring_last_generated_date DATE NULL
- parent_recurring_id INT NULL  -- 指向定期交易规则
```

### 关系
```
RecurringTransaction (规则)
    ↓ generates
Transaction (实际交易记录)
    - parent_recurring_id → RecurringTransaction.id
```

## 用户体验流程

### 场景1：手动创建定期交易
```
1. 点击"添加交易"
2. 填写基本信息
   - 描述：健康保险
   - 金额：120
   - 类型：支出
   - 类别：保险
3. 选择"定期交易"
4. 设置定期参数
   - 频率：每月
   - 开始：2024-01-01
5. 点击"保存"
6. 系统自动：
   - 创建定期规则
   - 生成当前月份的交易记录
   - 设置下次生成日期
```

### 场景2：OCR识别创建
```
1. 上传租赁合同
2. OCR识别完成
3. 显示建议：
   "检测到月租金 €640，是否创建定期交易？"
4. 点击"确认创建"
5. 自动填充表单（可修改）
6. 保存后自动生成
```

### 场景3：管理定期交易
```
1. 在交易列表中
2. 点击筛选"🔄 定期"
3. 看到所有定期交易
4. 点击某个定期交易
5. 可以：
   - 暂停（不再自动生成）
   - 编辑（修改金额、频率等）
   - 删除（删除规则和未来交易）
   - 查看历史（已生成的交易）
```
