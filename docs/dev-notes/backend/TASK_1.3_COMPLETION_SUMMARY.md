### 1. Transaction Model Updates (`backend/app/models/transaction.py`)

#### Added Fields:

**property_id** (UUID, nullable)
- Type: `UUID(as_uuid=True)`
- Nullable: Yes (not all transactions are property-related)
- Foreign Key: References `properties.id`
- On Delete: `SET NULL` (preserves transaction if property is deleted)
- Index: Yes (for query performance)

**is_system_generated** (Boolean, not nullable)
- Type: `Boolean`
- Nullable: No
- Default: `False`
- Purpose: Flag depreciation transactions auto-generated by the system

#### Added Relationship:
```python

```

## Usage Examples

### Linking a Transaction to a Property

```python
from app.models.transaction import Transaction, TransactionType, IncomeCategory
from decimal import Decimal
from datetime import date

# Create rental income transaction linked to property
transaction = Transaction(
    user_id=user.id,
    property_id=property.id,  # Link to property
    type=TransactionType.INCOME,
    amount=Decimal("1200.00"),
    transaction_date=date(2026, 3, 1),
    description="Rental income - March 2026",
    income_category=IncomeCategory.RENTAL,
    is_deductible=False,
    is_system_generated=False
)
db.add(transaction)
db.commit()
```

### System-Generated Depreciation Transaction

```python
from app.models.transaction import ExpenseCategory

# Create system-generated depreciation transaction
depreciation = Transaction(
    user_id=user.id,
    property_id=property.id,
    type=TransactionType.EXPENSE,
    amount=Decimal("5600.00"),
    transaction_date=date(2025, 12, 31),
    description=f"AfA {property.address} (2025)",
    expense_category=ExpenseCategory.DEPRECIATION,
    is_deductible=True,
    is_system_generated=True  # Flag as system-generated
)
db.add(depreciation)
db.commit()
```

### Querying Property Transactions

```python
# Get all transactions for a property
property_transactions = db.query(Transaction).filter(
    Transaction.property_id == property_id
).all()

# Get only manual transactions (exclude system-generated)
manual_transactions = db.query(Transaction).filter(
    Transaction.property_id == property_id,
    Transaction.is_system_generated == False
).all()

# Access property from transaction
transaction = db.query(Transaction).filter(Transaction.id == transaction_id).first()
property_address = transaction.property.address  # Via relationship
```

## Integration with Property Model

The Property model already has the reciprocal relationship defined:

```python
# In backend/app/models/property.py
transactions = relationship("Transaction", back_populates="property", foreign_keys="Transaction.property_id")
```

This enables bidirectional navigation:
- From Transaction → Property: `transaction.property`
- From Property → Transactions: `property.transactions`
